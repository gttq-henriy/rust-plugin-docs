# DynamicMonuments プラグイン 使用方法

## 概要
サーバー上に動的なモニュメント（建造物セットなど）を自動・手動でスポーンさせることができるプラグインです。
あらかじめ定義されたプレハブ（Prefab）を用いて、木箱、NPC、カードドアなどをまとめて配置し、ルートテーブル（アイテムドロップ）の制御管理なども統合的に行えます。

## 主な機能
- **自動スポーン機能**: 設定した場所に自動的にモニュメントを出現・リスポーンさせます。
- **グレネード/フレアによるサモン**: 指定のスキンが適用されたフレアを投げることで、プレイヤーがその場にモニュメントを召喚できます。
- **カードドア / エレベーター対応**: モニュメント内にアクセス制限のあるカードドアや、カスタムエレベーターを組み込むことができます。
- **NpcSpawn / カスタムルート（Loot）連携**: `NpcSpawn`, `AlphaLoot`, `CustomLoot`, `Loottable` プラグインと連携して、モニュメント内の木箱や出現するNPC（例：Scientist）のドロップアイテムを細かく制御できます。

## コンソール・チャットコマンド
以下のコマンドはすべて**管理者権限（Admin）専用**です。一般プレイヤーには使用できません。チャット(` / `) またはコンソールとして入力します。

| コマンド | 引数 | 説明 |
|---|---|---|
| `spawnmonumentmypos` | `[preset_name]` | 自分（管理者）が立っている位置に指定したモニュメントをスポーンさせます。 |
| `spawnmonument` | `[preset_name]` | （チャット/コンソール両対応）指定したモニュメントをスポーンさせます。 |
| `killmonument` | （なし） | 視線の先にあるモニュメント（エンティティ）を削除します。 |
| `killallmonuments` | （なし） | サーバー上の全てのDynamicMonumentsを削除します。 |
| `respawnmonuments` | （なし） | 自動スポーン設定されている全てのモニュメントをリスポーンさせます。 |
| `givemonument` | `[preset_name] <player_id>` | 指定したモニュメントを召喚できるアイテム（フレア等）をプレイヤーに付与します。 |
| `removemonumententity` | （なし） | 視線の先にある個別のモニュメント関連エンティティを削除します。 |
| `addprefabspawnpoint` | `[preset_name]` | 現在のモニュメントに新しいプレハブ（クレート等の）湧き位置を追加します。 |
| `addgroundnpcspawnpoint`| `[preset_name]` | 現在のモニュメントに新しいNPC（地上歩行）湧き位置を追加します。 |
| `replacecrate` | `[preset_name]` | 視線の先にあるクレート（木箱など）を、指定したプリセットのクレートに置き換えます。 |

## 補足
- 召喚用のフレア等は `givemonument` やショップ等経由でプレイヤーに配布します。
- カスタムモニュメント内では、ハック行為（FlyHack等）の判定が部分的にバイパスされるような処理（ゾーン制御）が行われています。
- `AlphaLoot` や `Loottable` との連携設定は、コンフィグ（設定ファイル）の `LootTableConfig` から設定可能です。
