# CustomSpawnPoints プラグイン 使用方法

## 概要
Rustの標準の初期スポーン地点（ランダムな海岸など）を無効化し、**管理者が独自に設定したポイント**の中からランダムにプレイヤーをスポーンさせるプラグインです。このプラグインは単体では動作せず、座標データの取得に **Spawns** プラグインを利用します。

## ターゲットユーザー
このプラグインは **サーバー管理者専用** です。一般プレイヤーが直接使用する機能やチャットコマンドはありません。（プレイヤーがベッド以外でリスポーンした際、自動的に設定されたポイントにスポーンします）

## 前提条件・連携プラグイン
- **Spawns**: 事前にこのプラグインを使用してスポーンポイント郡（ファイル）を作成しておく必要があります。

## サーバー管理者向けの使用方法
### 導入と設定の流れ
1. まず事前に `Spawns` プラグインを使用して、スポーンさせたい場所のファイル（例: `custom_beach`）を作成・保存しておきます。
2. `CustomSpawnPoints.cs` を `carbon/plugins/` に配置してリロードします。
3. サーバーコンソールまたはゲーム内（管理者権限が必要）から以下のコマンドでスポーンファイル名を指定します。

### コマンド（Console / RCON）
- `spawns.config <ファイル名>` : 初期スポーンに使用するSpawnsのファイル名を設定します。  
  例: `spawns.config custom_beach`  
  ※設定が完了するとそのファイル内のポイント群が読み込まれ、以降のプレイヤーのリスポーンはそこに限定されます。

### 設定ファイル (`carbon/config/CustomSpawnPoints.json`)
コンソールコマンドで変更する代わりに、直接ファイルを編集することも可能です。
- `"Spawnfile name"`: 使用するスポーンファイルの名前（拡張子なし）。空欄の場合は動作しません。
- `"Entity detection radius"`: スポーン位置の安全確認（建築物や障害物がないか検出する）半径。デフォルトは `10` です。

### 補足
リスポーン時に、選ばれた地点の周囲 `Entity detection radius` 範囲内に他プレイヤーの建築物や設置物（壁、チェストなど）がある場合、プラグインはスタックやリスキルを防ぐためにそこへのスポーンをキャンセルし、リスト内の別の候補地点を再選択します。
