# Spawns プラグイン 使用方法

## 概要
ユーザーがカスタムのスポーンポイント（復活や出現の位置）のセットを作成・保存できるデータベースプラグインです。このプラグイン自体が何かをスポーンさせるわけではなく、「BossMonster」や「RobSpawn」などの**他のプラグインから座標リストとして参照・利用されるための土台（API）**として機能します。

## ターゲットユーザー
このプラグインは **サーバー管理者専用** です。（AuthLevel 1以上の権限が必要です）
一般プレイヤーが直接使用するコマンドや機能はありません。主に他のイベントプラグインなどで指定するスポーン位置を定義するために使用されます。

## サーバー管理者向けの使用方法
管理者はチャットコマンドを使用して自身の現在位置をスポーンポイント群（スポーンファイル）として記録し、名前を付けて保存します。

### コマンド
- `/spawns new` : 新しいスポーンファイルの作成を開始します。
- `/spawns open <ファイル名>` : 編集するために既存のスポーンファイルを開きます。
- `/spawns add` : **現在の立ち位置**を新しいスポーンポイントとして追加し、画面上に緑色のボックスを描画して確認します。
- `/spawns remove <番号>` : 指定した番号のスポーンポイントを削除します。
- `/spawns save <ファイル名>` : 作成・編集したポイント群に名前を付けて保存します。
- `/spawns show [表示秒数]` : 現在作成中のポイントすべてに緑色のボックスを描画して視覚的に確認します。（デフォルトは10秒間表示）
- `/spawns close` : 保存せずに作成・編集をキャンセルして終了します。

### 作業の流れの例
1. スポーンさせたいエリアに行きます。
2. `/spawns new` を入力して作成を開始します。
3. 位置を少しずつ変えながら `/spawns add` を入力し、複数のポイントを登録します。
4. `/spawns save custom_arena` のように名前を付けて保存します。
5. 他のプラグインの設定ファイルで、スポーンファイル名として `custom_arena` を指定します。

### プラグインの導入
1. `Spawns.cs` を `carbon/plugins/` に配置します。
2. ロード: `carbon.reload Spawns`
3. 以降、作成したスポーンファイルは `carbon/data/SpawnsDatabase/` フォルダ内に保存されます。
