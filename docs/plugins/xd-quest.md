# XDQuest プラグイン 使用方法

## 概要
サーバー内に高機能な「クエストシステム」を導入するプラグインです。プレイヤーはクエストUIまたは専用NPCからクエスト（採集、キル、クラフトなど多様な目標）を受注し、達成することで様々な報酬（アイテム、設計図、カスタムコマンド実行など）を獲得できます。クエストNPCがいる専用の建物をマップ上に自動配置する機能も備えています。

## 操作方法（プレイヤー向け）
| コマンド | 説明 |
|---|---|
| `/qlist` または `/quest` | （チャットコマンド）クエストのリストと進行状況を確認できる専用UIを開きます。（※コマンド名は設定で変更可能） |

### UIからできること
- **クエストの受注・破棄**: 自分が受けたいクエストを選んで開始したり、進行中のものを破棄したりできます。
- **クエストの報告（完了）**: 必要なアイテムを集めるなどのクエスト条件を満たした後、UIから「完了（DELIVER / COMPLETE）」ボタンを押して報酬を受け取ります。
- **進行状況の確認**: 討伐数や採集数などの進捗がリアルタイムで確認できます。

## 管理者向け機能・仕様
| 管理コマンド | 説明 |
|---|---|
| `/quest.saveposition` | （チャットコマンド）現在位置をNPC（またはクエストルーム建物）のカスタムスポーン位置として保存します。 |
| `/quest.saveposition.outpost` | （チャットコマンド）アウトポスト（セーフゾーン）を基準とした相対位置として設定を保存します。 |
| `xdquest.player.reset [SteamID64]` | （コンソールコマンド）指定したプレイヤーのクエスト進行状況やクールダウンをリセットします。 |

### クエストの種類 (QuestType)
設定ファイルから作れるクエストの目的には以下のようなものがあります。
- `Gather` (採集)
- `EntityKill` (キル対象: 動物, NPC, プレイヤー等)
- `Craft` (クラフト) / `Loot` (ルート・箱開け)
- `RaidableBases` (RaidableBases攻略) / `Fishing` (釣り)
- 対象のアイテムを納品する（`Delivery`, `PurchaseFromNpc`など）
- その他多数（イベント進行系など）

### 設定（Config）について
`oxide/config/XDQuest.json` から以下の設定が可能です。
- **SpawnSettings**: クエストルーム（建物）を自動スポーンするか、NPC単体をスポーンさせるか、セーフゾーンを設けるかを選択できます。
- **UI等の表示**: HUD通知(`Notify`連携)や `IQChat` を使ったチャット装飾、UI上のアイコンや画像URLを設定できます。
- **Discord連携**: `discordWebhookUrl` を設定することで、サーバー内で完了したクエストの統計情報（Top5など）をDiscordへ定期配信できます。

### クエストデータの設定
実際のクエストリスト（内容や報酬）は `oxide/data/XDQuest/Quest.json` 等に保存・作成されます（デフォルトでプラグインが読み込みます）。Webサイト上のエディタ（https://xdquest.skyplugins.ru/）を使ってクエストのJSONファイルを生成することも推奨されています。
